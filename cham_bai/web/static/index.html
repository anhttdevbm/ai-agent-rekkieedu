Chấm bài

Đề bài: đường dẫn file .docx trên máy chạy server hoặc link Google Docs (chia sẻ xem được). Bài nộp: mỗi dòng một thư mục hoặc link GitHub (có thể để trống dòng nếu dòng đó chỉ chấm repo báo cáo). Báo cáo (tuỳ chọn): mỗi dòng một link GitHub hoặc Google Docs — dòng 1 khớp bài nộp dòng 1; có thể chỉ điền báo cáo (Docs) mà không điền bài nộp. Repo private: đặt biến môi trường GITHUB_TOKEN.

Đăng nhập trang quản lý để lấy token (tuỳ chọn)

Token chỉ lưu tạm trên trình duyệt (không gửi khi chấm bài).

Chỉ hiện session có type = THỰC HÀNH. Chọn xong sẽ tự điền “Đề bài” bằng link miniProject (nếu có).

Nguồn: /practice-resource?classId=&sessionId=. Chọn Lớp + Session ở trên rồi bấm tải.

Chọn Mã SV Họ tên Link Git Link báo cáo Điểm (nếu có)

Ví dụ chấm 20 bài: hệ thống sẽ tự chia lô để tránh timeout.

Tăng số này sẽ nhanh hơn nhưng dễ quá tải server/rate-limit.


        

      


      
      

        
Tạo Quiz (Excel)

        

          Không upload Excel: Quizz đầu giờ — theo session → mẫu quiz trong package (1 hàng/câu).
          Quizz Session đầu giờ và Quizz Session cuối giờ → mặc định
          example/Quizz_Session_Dau_Gio_Example.xlsx
          (đúng 45 câu; đầu giờ dùng difficulty 4/5/6/7/8/9; cuối giờ dùng 6/10/11).
          Tài liệu bài giảng: dùng Google Docs (khuyến nghị) để tránh tạo câu hỏi ngoài chương trình.
        

        

          Phân bổ 45 câu — Session đầu giờ
          

            45 câu: 30 câu BÀI CŨ + 15 câu BÀI MỚI.
            Difficulty chỉ dùng 6 mức và chia đều:
            4 Vận dụng chuyên sâu,
            5 Phân tích chuyên sâu,
            6 Sáng tạo,
            7 Thông hiểu,
            8 Vận dụng sơ bộ,
            9 Phân tích sơ bộ.
            (Mỗi mức 7 câu; 3 mức đầu được +1 câu.)
          

        

        

          Phân bổ 45 câu — Session cuối giờ
          

            45 câu: chỉ lấy session hiện tại. Difficulty chia đều:
            6 Sáng tạo,
            10 Vận dụng,
            11 Phân tích (mỗi mức 15 câu).

Chấm bài Nhóm A

        

          Dán link Google Docs đề bài và danh sách sinh viên (mỗi dòng: Họ tên[TAB]Mã SV[TAB]Link repo).
          Hệ thống sẽ chấm theo lô và trả về điểm + nhận xét chi tiết (4–8 câu).
        

        

          

            
            
          

          

            
            
            

              Nếu không có TAB, hệ thống sẽ hiểu: repo là token cuối, mã là token áp cuối, còn lại là họ tên.
